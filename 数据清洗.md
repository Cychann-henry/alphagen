### 第二部分：一键启动脚本 (Command File)

为了方便你管理实验参数（利用好你的 4090），我把启动命令封装成了一个 Shell 脚本。

请在项目根目录（`root/ccc/alphagen`）下创建一个新文件，命名为 **`run_training.sh`**。

#### 1. 脚本内容 (复制以下代码)

```bash
#!/bin/bash

# ================= 配置区域 (在这里修改参数) =================

# 1. 实验名称 (建议每次改名，如 run_01, run_02, run_high_batch)
EXP_NAME="run_01_baseline"

# 2. 指定显卡 (0, 1, 2, 3)
# 单卡跑设为 "0"，想换卡跑设为 "1"
GPU_ID="0"

# 3. 随机种子 (复现关键，想跑差异化结果就改这个，如 1, 42, 1024)
SEED=1

# 4. 显卡参数 (针对 4090 优化)
# 4090 显存大，建议 2048 ~ 8192。太小浪费，太大可能影响收敛。
BATCH_SIZE=4096

# 5. 训练总步数 (步数越多，见多识广)
# 建议至少 1,000,000 (一百万步)
STEPS=2000000

# 6. 路径设置 (通常不用动)
SAVE_PATH="./experiments/${EXP_NAME}"
LOG_PATH="${SAVE_PATH}/tb_logs"

# ==========================================================

# 打印一下当前配置，方便确认
echo "----------------------------------------------------"
echo "🚀 启动 AlphaGen 训练任务: ${EXP_NAME}"
echo "----------------------------------------------------"
echo "📌 使用显卡 (GPU): ${GPU_ID}"
echo "📌 随机种子 (Seed): ${SEED}"
echo "📌 Batch Size: ${BATCH_SIZE}"
echo "📌 训练步数: ${STEPS}"
echo "📂 结果保存路径: ${SAVE_PATH}"
echo "----------------------------------------------------"

# 激活 Conda 环境 (防止你在 base 环境里裸跑)
# 注意：你需要把 'alphagen_env' 换成你真实的环境名
# 如果你在终端已经激活了，下面这两行可以注释掉
source ~/miniconda3/etc/profile.d/conda.sh 2>/dev/null || source ~/.bashrc
conda activate alphagen_env 2>/dev/null || echo "⚠️  请确保已激活 Conda 环境！"

# 创建目录 (防止报错)
mkdir -p "${LOG_PATH}"

# 设置显卡可见性 (物理隔离)
export CUDA_VISIBLE_DEVICES=${GPU_ID}

# 执行训练命令
# python -m 是标准启动方式
python -m scripts.rl \
    --device cuda:0 \
    --seed ${SEED} \
    --save_path "${SAVE_PATH}" \
    --tensorboard_log "${LOG_PATH}" \
    --batch_size ${BATCH_SIZE} \
    --steps ${STEPS}

echo "----------------------------------------------------"
echo "✅ 训练结束！请使用 Tensorboard 查看日志："
echo "tensorboard --logdir ${LOG_PATH}"
echo "----------------------------------------------------"

```

#### 2. 如何使用这个脚本

1. **创建文件**：
在 VS Code 左侧文件栏右键 -> New File -> `run_training.sh` -> 粘贴上面的内容。
2. **赋予执行权限** (在终端运行一次即可)：
```bash
chmod +x run_training.sh

```


3. **运行脚本**：
```bash
./run_training.sh

```



### 第三部分：接下来的操作建议

1. **试运行**：
保持脚本里默认的参数（或者把 `STEPS` 改小一点，比如 `10000`），运行 `./run_training.sh`，看能不能跑通全流程且不报错。
2. **正式运行**：
* 把 `STEPS` 改回 `2000000`。
* 运行 `./run_training.sh`。
* 利用 `nohup` 后台运行（防止断网掉线）：
```bash
nohup ./run_training.sh > output.log 2>&1 &

```




3. **多卡利用**：
当你熟练后，你可以复制这个文件为 `run_gpu1.sh`，把里面的 `GPU_ID="0"` 改成 `"1"`，把 `SEED=1` 改成 `2`，然后同时运行这两个脚本，就能双倍效率挖掘因子了。

准备好就开始吧！
